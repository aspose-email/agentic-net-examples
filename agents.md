## Boundaries
### ✅ Always
- Use explicit types for Aspose objects when clarity is needed.
- Dispose `SmtpClient`, `ImapClient`, `Pop3Client`, streams, and messages with `using` where applicable.

### ⚠️ Ask First
- Adding new NuGet dependencies besides `Aspose.Email`.
- Changing shared templates or pipeline settings.

### 🚫 Never
- Create multi-file projects; keep each example in one `.cs` file.
- Hardcode credentials, secrets, or license keys.
- Omit required namespaces—prefer fully qualified names if ambiguity arises.

## Testing Guide
- **Build**: `dotnet build --configuration Release --verbosity minimal`
- **Run**: `dotnet run`
Success = exit code 0 and no `CS####` compiler errors.

## Domain Knowledge (Email-specific)
- Use `MailMessage`/`MapiMessage` appropriately for MIME vs MAPI scenarios.
- Prefer `SecurityOptions.Auto` for SMTP/IMAP/POP TLS negotiation.
- Save MSG with `SaveOptions.DefaultMsgUnicode` to preserve UTF-8 content.
